# Git Commit 컨벤션
## Rule

- 커밋 메시지는 한글로 작성한다.
- 커밋 Type은 대문자로 시작한다.

### 커밋 Type
```
------------------------------------------------------------------
Feat : 새로운 기능 추가
Fix : 버그 수정
Docs : 문서 수정
Refactor : 코드 리팩토링, 세미콜론 누락, 코드 변경이 없는 경우
Chore : 분류하기 어려운 자잘한 수정
------------------------------------------------------------------
```

### 예시
```
Feat: todos 앱 ~~ URL 라우팅 추가
Fix: 버튼 클릭시 ~~ 문제 수정
```


### Pull Request

- **모두 Approve**를 해야 머지가 가능하다. (어차피 2명임)
